# 11 テスト仕様書

## 1. テスト方針

本プロジェクトのテストは以下の方針で行う。

- ビジネスロジック（RAG Service・LLM Adapter・Vector Store Adapter）は**単体テスト必須**
- 外部依存（LLM API・Supabase・OpenAI Embeddings API）は**モックで切り離す**
- API エンドポイントは **TestClient を使った統合テスト**を行う
- バッチ処理（Ingest Pipeline）は**単体テスト必須**（外部依存はモック）
- フロントエンド（Next.js）のテストは MVP スコープ外（将来追加）

---

## 2. テストフレームワーク・ツール

| 対象 | ツール | 理由 |
|---|---|---|
| Python（FastAPI・バッチ） | pytest | Python の標準的なテストフレームワーク |

## 4. モック戦略

### 4.1 外部依存の切り離し方針

外部依存は以下の 3 種類に分類し、それぞれモック化する。

| 外部依存 | モック手法 | 理由 |
|---|---|---|
| Gemini API（LLM） | `FakeLLM` クラスで `BaseLLMClient.generate` をスタブ（呼び出し引数を記録） | LLM は非決定的で課金が発生するため常にモックする |
| OpenAI Embeddings API | `unittest.mock.patch` で `app.services.embedder.AsyncOpenAI` を差し替える | API コストを発生させずに embedding を再現する |
| asyncpg（PostgreSQL） | `asyncpg.create_pool` を `AsyncMock` で差し替え、`Pool.acquire()` をコンテキストマネージャ Mock にする | テスト用 DB を用意せずにロジックをテストする |

### 4.2 conftest.py の共通フィクスチャ

統合テスト用の `TestClient`（同期）を `_make_test_client()` ジェネレーターで生成する。
起動時に `asyncpg.create_pool` と `_create_llm_client` をパッチし、テスト後にキャッシュをクリアする。

```python
# tests/conftest.py
from unittest.mock import AsyncMock, MagicMock, patch
import pytest
from fastapi.testclient import TestClient
import app.main as _main
from app.main import app
from app.settings import get_settings

_FAKE_ENV = {
    "DATABASE_URL": "postgresql://test:test@localhost/test",
    "OPENAI_API_KEY": "sk-test",
    "GEMINI_API_KEY": "gemini-test",
    "LLM_PROVIDER": "gemini",
}

class _FakeLLM:
    def __init__(self): self.calls = 0
    async def generate(self, system_prompt, user_message, **kwargs):
        self.calls += 1
        return "ok"

def _make_test_client(db_ok=True, llm=None):
    """TestClient を生成するジェネレーター（pytest fixture として使用）"""
    mock_conn = AsyncMock()
    mock_conn.fetchval = AsyncMock(return_value=1) if db_ok \
        else AsyncMock(side_effect=Exception("connection refused"))
    cm = AsyncMock()
    cm.__aenter__ = AsyncMock(return_value=mock_conn)
    cm.__aexit__ = AsyncMock(return_value=None)
    mock_pool = MagicMock()
    mock_pool.acquire.return_value = cm
    mock_pool.close = AsyncMock()
    llm = llm or _FakeLLM()
    with patch.dict("os.environ", _FAKE_ENV):
        get_settings.cache_clear()
        get_settings()
        with (
            patch("app.main.asyncpg.create_pool", AsyncMock(return_value=mock_pool)),
            patch.object(_main, "_create_llm_client", return_value=llm),
            patch("app.services.embedder.AsyncOpenAI"),
        ):
            with TestClient(app) as client:
                client.app.state.test_llm = llm
                yield client
    for key in ("llm_health_status", "llm_health_checked_at", "test_llm"):
        if hasattr(app.state, key):
            delattr(app.state, key)
    get_settings.cache_clear()

@pytest.fixture
def test_client(): yield from _make_test_client()

@pytest.fixture
def db_error_client(): yield from _make_test_client(db_ok=False)

@pytest.fixture
def llm_error_client():
    class _FailingLLM:
        def __init__(self): self.calls = 0
        async def generate(self, *args, **kwargs):
            self.calls += 1
            raise RuntimeError("llm unavailable")
    yield from _make_test_client(llm=_FailingLLM())
```

## 5. 単体テスト仕様

### 5.1 RAG Service テスト

| テストID | テスト内容 | 前提条件 | 期待結果 |
|---|---|---|---|
| RAG-01 | 正常な質問で回答と出典が返る | モック LLM・モック Vector Store | answer が文字列、sources が 1 件以上 |
| RAG-02 | 検索結果が 0 件の場合 | Vector Store が空リストを返す | 固定文を返し、sources は空、LLM は呼ばれない |
| RAG-03 | LLM がエラーを返した場合 | LLM が例外を raise | `HTTPException(500, LLM_ERROR)` が raise される |
| RAG-04 | top_k パラメータが正しく渡される | top_k=3 で呼び出す | Vector Store に top_k=3 が渡されている |
| RAG-05 | コンテキストが正しく組み立てられる | チャンク 2 件 | プロンプトに両チャンクのテキストが含まれる |
| RAG-06 | 競技名なしで複数競技が検索される | ホッケー・空手の検索結果 | 競技名の指定を促し、LLM は呼ばれない |
| RAG-07 | 競技名ありで複数競技が検索される | 質問に「ホッケー」を含む | ホッケー文書だけを LLM に渡す |
| RAG-08 | 指定競技に一致する検索結果がない | 質問に「空手」、結果はホッケーのみ | 指定競技の情報がない旨を返し、LLM は呼ばれない |
| RAG-09 | 類似度が低い検索結果のみ | similarity < 0.65 | 固定文を返し、LLM は呼ばれない |

単体テストでは `FakeLLM`（呼び出し引数を記録）と `FakeVectorStore`（固定データを返す）を使いコンストラクタ注入する。

```python
# tests/unit/test_rag_service.py（抜粋）
class FakeLLM:
    def __init__(self, return_value="コンテキストに基づく回答です。"):
        self.last_system_prompt = ""
        self.last_user_message = ""
        self.return_value = return_value

    async def generate(self, system_prompt, user_message, **kwargs):
        self.last_system_prompt = system_prompt
        self.last_user_message = user_message
        return self.return_value

@pytest.mark.asyncio
async def test_chat_returns_answer_and_sources():
    vector_store = FakeVectorStore()
    llm = FakeLLM()
    service = RAGService(FakeEmbedder(), vector_store, llm)
    response = await service.chat("オフサイドとは？", "session-1", 5)

    assert response.answer == "コンテキストに基づく回答です。"
    assert response.sources[0].title == "FIFA サッカー競技規則"
    assert "オフサイドに関する説明" in llm.last_system_prompt  # コンテキスト注入を確認

@pytest.mark.asyncio
async def test_chat_empty_search_results_returns_fixed_message_without_llm():
    vector_store = FakeVectorStore(results=[])
    llm = FakeLLM()
    service = RAGService(FakeEmbedder(), vector_store, llm)
    response = await service.chat("存在しない質問", "session-1", 5)

    assert response.answer == "提供された情報の中には該当する情報が見つかりませんでした。"
    assert response.sources == []
    assert llm.last_user_message == ""  # LLM が呼ばれていないことを確認
```
