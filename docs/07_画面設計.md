### 2.3 出典表示（SourceBadge）の設計

出典は回答テキストの下部に「【出典】」セクションとして表示する。

#### 表示項目

| 項目 | 表示内容 | 例 |
|---|---|---|
| 信頼性ラベル | authority_score に応じたバッジ | ⚡高信頼 / ✓中信頼 / △要確認 |
| タイトル | documents.title | FIFA サッカー競技規則 2024/25 |
| 組織名 | documents.organization | FIFA |
| リンク | documents.source_url（外部リンク） | [→] ボタン |
| 類似度 | similarity スコア（任意表示） | 87% |

#### 信頼性バッジの色分け

| authority_score | ラベル | 色 | アイコン |
|---|---|---|---|
| 0.80 以上 | 高信頼 | 緑（#16a34a） | ⚡ |
| 0.50 〜 0.79 | 中信頼 | 青（#2563eb） | ✓ |
| 0.50 未満 | 要確認 | 黄（#d97706） | △ |

色だけで判断させないよう、ラベルテキストを必ず併記する（アクセシビリティ対応）。

#### SourceBadge のUI仕様

```
┌──────────────────────────────────────────────────┐
│ [⚡高信頼]  FIFA サッカー競技規則 2024/25  [→]  │
│              FIFA  •  類似度 87%                 │
└──────────────────────────────────────────────────┘
```

- 複数出典は類似度の高い順に表示する
- 外部リンクは `target="_blank" rel="noopener noreferrer"` を設定する
- タイトルが長い場合は 40 文字で省略（ツールチップで全表示）

## 3. ローディング状態

回答生成中は以下のインジケータを表示する。

```
[...] 回答を生成しています（検索 → 生成の段階表示）
```

段階表示（オプション）:

1. `ドキュメントを検索しています...`
2. `回答を生成しています...`

## 4. エラー表示

| エラー種別 | 表示メッセージ |
|---|---|
| LLM 接続エラー | 「回答の生成に失敗しました。しばらく後にお試しください。」 |
| 検索結果なし | 「関連するドキュメントが見つかりませんでした。別の質問をお試しください。」 |
| ネットワークエラー | 「通信エラーが発生しました。インターネット接続を確認してください。」 |

エラーはインラインで表示し、モーダルは使わない（操作を妨げないため）。

## 5. アクセシビリティ設計

- チャット入力欄: `<label>` または `aria-label` を設定する
- 送信ボタン: アイコンのみの場合は `aria-label="送信"` を設定する
- ローディング状態: `aria-live="polite"` で読み上げ対応する
- 出典リンク: `aria-label="FIFA サッカー競技規則 2024/25 を開く（外部サイト）"` のように設定する
- カラーコントラスト: 全テキスト要素で 4.5:1 以上を確保する
- フォーカス順序: 入力欄 → 送信ボタン → 出典リンクの順を保つ

## 6. レスポンシブ対応

| ブレークポイント | 対応 |
|---|---|
| モバイル（< 768px） | 1カラム。出典は折りたたみ表示（デフォルト展開） |
| タブレット（768px〜） | 1カラム。余白を広げる |
| デスクトップ（1024px〜） | 最大幅 800px で中央揃え |
