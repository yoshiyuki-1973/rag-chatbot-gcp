# GCP 運用コスト見積もり

本システムを GCP（Google Cloud Platform）へ移行した際の、検証環境（Staging）および本番環境（Production）を維持・運用するための月額コスト見積もりです。

---

## 1. サービス別の月額コスト内訳（目安）

### 1.1 Cloud SQL for PostgreSQL (データベース/ベクトルDB)
Cloud SQL はサーバーレスではないため、起動時間に応じた**固定費**が発生します。

- **スペック**: `db-f1-micro` (1 vCPU, 0.6 GB RAM) - 最小構成
- **稼働時間**: 24時間365日
- **基本料金（東京リージョン）**: 約 $9.50 / 月
- **ストレージ（10 GB SSD）**: 約 $1.70 / 月
- **合計**: **約 $11.20 / 月（約 1,750 円 / 台）**
  - 検証環境 (`stg`) と 本番環境 (`prod`) で合計2台運用する場合: **約 $22.40 / 月（約 3,500 円）**

### 1.2 Cloud Run (Next.js フロントエンド & FastAPI バックエンド)
Cloud Run はサーバーレスのため、アクセス（リクエスト）が発生した時間分のみの**従量課金**です。

- **アクセス規模の想定**: 月間 10,000 リクエスト（1日約330リクエスト）
  - vCPU / メモリ秒の消費は無料枠（18万 vCPU秒、36万 GiB秒）に完全に収まります。
- **コールドスタート対策（最小インスタンス数設定）**:
  - **`min-instances = 0` の場合 (検証環境・個人開発推奨)**: アクセスがない時は 0 インスタンスになるため **$0 / 月**。
  - **`min-instances = 1` の場合 (本番環境推奨)**: コールドスタート（起動待ち）を防ぐため、バックエンドのみ常時1インスタンス起動させた場合、**約 $10.00 〜 $12.00 / 月（約 1,600 〜 1,900 円）**。

### 1.3 Vertex AI (Gemini 2.5 Flash & text-embedding-004)
API 呼び出しのトークン量に応じた**従量課金**です。Vertex AI の Gemini 2.5 Flash は非常に低コストに設計されています。

- **LLM (Gemini 2.5 Flash)**:
  - インプット: $0.000075 / 1,000 トークン
  - アウトプット: $0.000300 / 1,000 トークン
  - *想定*: 1回あたり平均 5,500 トークン消費（インプット 5,000 + アウトプット 500）、月 1,000 回のチャット。
  - **計算**: `((5.0 * $0.000075) + (0.5 * $0.000300)) * 1,000回 = $0.525 / 月 (約 80 円)`
- **Embedding (text-embedding-004)**:
  - ベクトル化: $0.000025 / 1,000 トークン
  - *想定*: 1回あたり平均 100 トークン、月 1,000 回の検索クエリ。データ登録時（インジェスト）に計 1,000 チャンク（約50万トークン）を処理。
  - **計算**: `(($0.000025 * 100) * 1,000回) + ($0.000025 * 500) = $0.038 / 月 (約 6 円)`
- **合計**: **約 $0.56 / 月（約 90 円）**

### 1.4 その他 (Artifact Registry / ネットワーク等)
- **Artifact Registry (イメージ保管)**: 5 GB まで無料枠が適用されるため、本システムのイメージ容量（合計約1GB程度）であれば **$0 / 月**。
- **Egress (下りネットワーク)**: 月 100 GB 未満のデータ送信であれば無料枠または数セント程度（**$0 / 月**）。

## 2. 運用プラン別月額シミュレーション

利用目的と予算に応じて、以下の2つのプランから選択できます。

### パターンA: 超節約プラン（個人開発・検証用途）
すべての環境でコストを最小限に抑え、アクセスがない時間帯の課金を完全に防ぐ構成です。

- **前提条件**:
  - 検証環境 (`stg`) と本番環境 (`prod`) の両方を運用。
  - Cloud Run はコールドスタート（最初のアクセス時に10〜15秒の待ち時間）を許容する (`min-instances = 0`)。
  - Cloud SQL は 2 環境とも最小構成 (`db-f1-micro`)。
- **月額コスト**:
  - Cloud SQL (2台) 固定費: 約 $22.40
  - Cloud Run (stg / prod) 従量費: $0.00 (無料枠内)
  - Vertex AI (API利用料): 約 $0.56
  - **月額合計: 約 $23.00（約 3,600 円 / 月）**

### パターンB: 本番運用プラン（サービス公開用途）
本番環境でのユーザーの使いやすさ（コールドスタートの防止）を重視した構成です。

- **前提条件**:
  - 本番環境 (`prod`) のバックエンド API のみ、常時1つのコンテナを待機させる (`min-instances = 1`)。これにより起動の遅延がなくなります。
  - 検証環境 (`stg`) の Cloud Run は `min-instances = 0` のまま維持。
  - Cloud SQL は 2 環境とも最小構成 (`db-f1-micro`)。
- **月額コスト**:
  - Cloud SQL (2台) 固定費: 約 $22.40
  - Cloud Run (`prod` バックエンド常時起動): 約 $12.00
  - Vertex AI (API利用料): 約 $0.56
  - **月額合計: 約 $35.00（約 5,500 円 / 月）**

## 3. コストをさらに抑えるためのアドバイス

1. **検証環境 (Staging) のDB停止または省略**:
   - 検証環境の Cloud SQL を常時起動すると月約 1,750 円かかります。もし検証が一時的であるなら、検証を行わない時は Cloud SQL インスタンスを「停止」状態にする（※停止中もストレージ費用のみ数十円〜数百円かかりますが、インスタンス実行費は$0になります）、または `stg` 用のDBは作成せずローカル開発と本番環境のみの2環境構成にすることで、**月額約 1,700 円の節約**になります。
2. **確約利用割引 (CUD) の活用（将来）**:
   - サービスの規模が拡大し、より高スペックな Cloud SQL インスタンスや常時稼働の Compute リソースが必要になった場合は、1年または3年の契約（確約利用割引）を行うことで、**最大 20% 〜 37% の割引**を適用できます。
