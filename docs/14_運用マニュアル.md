# 14 運用マニュアル

## 1. デプロイ構成

### 1.1 全体デプロイ構成

| サービス | ホスティング先 | プラン | 用途 |
|---|---|---|---|
| Next.js フロントエンド | Vercel Hobby | 無料 | UI 配信・CDN |
| FastAPI バックエンド | Render Free | 無料 | API サーバー |
| PostgreSQL + pgvector | Supabase Free | 無料 | DB・ベクトル検索 |
| Ingest バッチ | ローカル Docker | — | データ取り込み |

---

### 1.2 Vercel（Next.js フロントエンド）

#### デプロイ手順

1. [vercel.com](https://vercel.com) にサインイン
2. 「Add New Project」→ GitHub リポジトリを選択
3. Framework Preset: **Next.js** を選択（自動検出される）
4. 環境変数を設定する

```
NEXT_PUBLIC_API_BASE_URL=https://<app-name>.onrender.com
```

5. 「Deploy」をクリック

#### 自動デプロイ

- `main` ブランチへのプッシュで本番環境に自動デプロイされる
- プルリクエスト作成でプレビュー環境が自動生成される

#### Vercel Hobby の制限事項

| 制限 | 内容 |
|---|---|
| ビルド時間 | 45 分/月 |
| 帯域幅 | 100 GB/月 |
| 商用利用 | 不可（ポートフォリオ用途は可） |

---

### 1.3 Render（FastAPI バックエンド）

#### デプロイ手順

1. [render.com](https://render.com) にサインイン
2. 「New Web Service」→ GitHub リポジトリを選択
3. 設定を入力する

| 設定項目 | 値 |
|---|---|
| Name | `rag-chatbot-api` |
| Region | `Oregon (US West)` |
| Branch | `main` |
| Root Directory | `backend` |
| Runtime | `Python 3.12` |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `uvicorn app.main:app --host 0.0.0.0 --port $PORT` |
| Instance Type | `Free` |

4. 環境変数を設定する

```
DATABASE_URL=postgresql://postgres.[project-ref]:...@aws-0-[region].pooler.supabase.com:5432/postgres（Supabase の Supavisor Session pooler 接続文字列）
GROK_API_KEY=xai-...
OPENAI_API_KEY=sk-...（embedding 用）
LLM_PROVIDER=grok
```

5. 「Create Web Service」をクリック

#### 自動デプロイ

- `main` ブランチへのプッシュで自動デプロイされる

#### Render Free の注意事項

| 制限 | 内容 | 対策 |
|---|---|---|
| スリープ | 15 分間リクエストがないとインスタンスがスリープ | 初回リクエスト時に 30〜50 秒の起動時間が発生する。フロントで「起動中...」表示を出す |
| CPU・メモリ | 0.1 CPU / 512MB RAM | 大量同時リクエストには対応できない（ポートフォリオ用途では許容） |
| 月次稼働時間 | 750 時間/月 | 1 インスタンスなら常時稼働可能 |

---

### 1.4 Supabase（PostgreSQL + pgvector）

#### セットアップ手順

1. [supabase.com](https://supabase.com) にサインイン
2. 「New Project」でプロジェクトを作成する
3. SQL Editor で pgvector 拡張を有効化する

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

4. テーブルを作成する（`docs/05_テーブルレイアウト.md` の DDL を参照）
5. 接続情報を取得する

```
Settings → Database → Connection string (URI)
postgresql://postgres.[project-ref]:[password]@aws-0-[region].pooler.supabase.com:5432/postgres
```

6. Row Level Security (RLS) の設定（任意・将来対応）
   - MVP では RLS を無効にし、サービスキーでのみアクセスする

#### Supabase Free の制限事項

| 制限 | 内容 |
|---|---|
| DB サイズ | 500 MB |
| 帯域幅 | 2 GB/月 |
| 非アクティブ停止 | 1 週間アクセスがないとプロジェクトが一時停止 → Supabase ダッシュボードから再開できる |

---

### 1.5 ローカル Docker（Ingest バッチ）

#### 前提条件

- Docker Desktop がインストール済みであること
- `.env` ファイルに必要な環境変数が設定済みであること

#### データ取り込み手順

1. データファイルを `data/` ディレクトリに配置する

```
data/
├── markdown/
│   └── joc_overview.md
└── pdf/
    └── fifa_report_2022.pdf
```

2. `sources.yaml` にメタ情報を追記する（`docs/06_RAG設計.md` §3.2 参照）

3. バッチを実行する

```bash
docker compose run --rm ingest
```

4. 実行ログでエラーがないことを確認する

```
[INFO] Processing: data/markdown/joc_overview.md
[INFO]   Extracted 1234 chars
[INFO]   Created 5 chunks
[INFO]   Generated embeddings
[INFO]   Upserted document: https://www.joc.or.jp/about/ (id: xxx)
[INFO]   Inserted 5 chunks
[INFO] Completed: 1/2 files processed, 0 files failed
```

## 2. 環境変数一覧

### バックエンド（FastAPI）

| 変数名 | 必須 | 説明 | 例 |
|---|---|---|---|
| DATABASE_URL | 必須 | Supabase PostgreSQL 接続文字列 | `postgresql://postgres.xxx:xxx@aws-0-region.pooler.supabase.com:5432/postgres` |
| GROK_API_KEY | 必須 | xAI Grok の API キー | `xai-xxx` |
| OPENAI_API_KEY | 必須 | OpenAI Embeddings API キー | `sk-xxx` |
| LLM_PROVIDER | 任意 | 使用する LLM プロバイダー | `grok`（デフォルト） |
| TOP_K_DEFAULT | 任意 | デフォルトの検索件数 | `5` |
| CORS_ORIGINS | 任意 | 許可するオリジン（カンマ区切り） | `https://xxx.vercel.app,http://localhost:3000` |

### フロントエンド（Next.js）

| 変数名 | 必須 | 説明 | 例 |
|---|---|---|---|
| NEXT_PUBLIC_API_BASE_URL | 必須 | バックエンド API の URL | `https://rag-chatbot-api.onrender.com` |

### バッチ（Ingest）

| 変数名 | 必須 | 説明 |
|---|---|---|
| DATABASE_URL | 必須 | Supabase PostgreSQL 接続文字列 |
| OPENAI_API_KEY | 必須 | OpenAI Embeddings API キー |

## 3. 監視・アラート

MVP では外部監視ツールは使用しない。以下の手動確認を定期的に行う。

| 確認項目 | 方法 | 頻度 |
|---|---|---|
| API 生存確認 | `GET /health` のレスポンスを確認 | 週 1 回 |
| Supabase 停止確認 | Supabase ダッシュボードにログインして確認 | 週 1 回 |
| Render ログ確認 | Render ダッシュボードのログタブを確認 | 問題発生時 |
| Vercel デプロイ確認 | Vercel ダッシュボードのデプロイ状態を確認 | プッシュ後 |

## 4. 将来の拡張

### 4.1 LLM の差し替え

`LLM_PROVIDER` 環境変数と `LLMAdapter` を追加するだけで切り替えられる。

| 対象 LLM | 変更点 |
|---|---|
| Claude（Anthropic） | `ClaudeLLMClient` を実装。`anthropic` SDK を使用。`messages` API の形式が若干異なるため Adapter で吸収する |
| OpenAI GPT-4o | `GrokLLMClient` の `base_url` を `https://api.openai.com/v1` に変更するだけ |
| ローカル Ollama | `base_url` を `http://localhost:11434/v1` に変更 |

### 4.2 ベクトル DB の差し替え

`VectorStoreAdapter` の実装を追加するだけで切り替えられる。

| 対象 | 実装クラス | 主な変更点 |
|---|---|---|
| Pinecone | `PineconeVectorStore` | `pinecone` SDK で `upsert` / `query` |
| Chroma | `ChromaVectorStore` | `chromadb` SDK で `add` / `query` |
| Weaviate | `WeaviateVectorStore` | `weaviate` SDK で操作 |

### 4.3 UI 改善

| 改善項目 | 内容 |
|---|---|
| ストリーミング回答 | LLM のレスポンスをトークン単位でストリーミング表示し、UX を向上させる |
| 会話履歴の活用 | chat_history テーブルから過去のターンを取得し、文脈を維持した会話を実現する |
| 検索フィルター | authority_score・organization でフィルターできる UI を追加する |
| ダークモード対応 | Tailwind CSS の `dark:` クラスを使ってダークモードに対応する |

### 4.4 認証の追加

- Supabase Auth（Google OAuth 等）でユーザー認証を追加する
- Row Level Security を有効にし、セッションをユーザー単位で管理する
- Vercel の環境変数に `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` を追加する

### 4.5 クラウドバッチ移行

データ更新が頻繁になった場合、以下を検討する。

- **GitHub Actions**: 定期実行（cron）で `ingest.py` を実行する
- **Cloud Run Jobs**: Google Cloud で Python バッチを実行する
- **Render Cron Jobs**: Render の有料プランで定期実行する
