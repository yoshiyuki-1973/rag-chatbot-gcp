## 2. バックエンドモジュール仕様

### 2.1 `app/main.py`

| 要素 | 種別 | 仕様 |
|---|---|---|
| `_create_llm_client(settings)` | 関数 | `settings.llm_provider` に応じて `BaseLLMClient` を生成するファクトリ。未知のプロバイダーは `RuntimeError` を送出する |
| `lifespan(app)` | 非同期コンテキストマネージャ | 起動時に必須環境変数チェック → `create_pool` → `EmbeddingClient` / `_create_llm_client` を `app.state` にセット。終了時に `db_pool.close()` を実行 |

### 2.2 `app/settings.py`

| フィールド | 型 | デフォルト | エイリアス（環境変数名） |
|---|---|---|---|
| `database_url` | `str \| None` | `None` | `DATABASE_URL` |
| `grok_api_key` | `str \| None` | `None` | `GROK_API_KEY` |
| `grok_model` | `str` | `"grok-4.20"` | `GROK_MODEL` |
| `openai_api_key` | `str \| None` | `None` | `OPENAI_API_KEY` |
| `llm_provider` | `str` | `"grok"` | `LLM_PROVIDER` |
| `top_k_default` | `int` | `5` | `TOP_K_DEFAULT` |
| `cors_origins` | `str` | `"http://localhost:3000"` | `CORS_ORIGINS` |

| バリデーター | 対象 | 規則 |
|---|---|---|
| `validate_top_k_default` | `top_k_default` | 1 以上 20 以下。範囲外は `ValidationError` |

| プロパティ | 型 | 説明 |
|---|---|---|
| `cors_origin_list` | `list[str]` | `cors_origins` をカンマ分割して返す |

### 2.3 `app/dependencies.py`

| 関数 | 戻り値 | 説明 |
|---|---|---|
| `get_rag_service(request)` | `RAGService` | `request.app.state` から依存オブジェクトを取り出し `RAGService` を生成する。`SupabaseVectorStore` はリクエストごとに新規生成（ステートレス） |

### 2.4 `app/services/rag_service.py`

**`RAGService` クラス**

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `__init__` | `(embedder, vector_store, llm)` | 依存オブジェクトをコンストラクタ注入 |
| `search` | `(query, top_k, min_similarity, min_authority_score) → SearchResponse` | クエリを embed してベクトル検索を実行 |
| `chat` | `(query, session_id, top_k) → ChatResponse` | 検索 → コンテキスト組み立て → LLM 生成 → 履歴保存の一連を実行 |
| `_build_context` | `(chunks: list[SearchResult]) → str` | チャンクリストをプロンプトのコンテキスト文字列に変換 |

**エラーハンドリング方針**

- 検索失敗（embedding・DB）: `HTTPException(500, SEARCH_ERROR)`
- LLM 失敗: `HTTPException(500, LLM_ERROR)`
- 履歴保存失敗: `logger.warning` のみ（回答は返す）

**定数**

```python
SYSTEM_PROMPT = """あなたはスポーツに関する質問に答えるアシスタントです。..."""
```

### 2.5 `app/services/embedder.py`

**`EmbeddingClient` クラス**

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `__init__` | `(api_key, model="text-embedding-3-small")` | `AsyncOpenAI` クライアントを初期化 |
| `embed` | `async (text: str) → list[float]` | テキストを 1536 次元ベクトルに変換 |

### 2.6 `app/adapters/llm/base.py`

**`BaseLLMClient` クラス（ABC）**

| メソッド | 種別 | シグネチャ | 説明 |
|---|---|---|---|
| `generate` | abstract | `async (system_prompt, user_message, max_tokens=1000, temperature=0.1) → str` | 回答テキストを返す |
| `stream` | default | `async (…) → AsyncIterator[str]` | `generate` の結果を 1 回 yield するデフォルト実装 |

### 2.7 `app/adapters/llm/grok.py`

**`GrokLLMClient` クラス（`BaseLLMClient` の実装）**

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `__init__` | `(api_key: str, model: str)` | `base_url="https://api.x.ai/v1"` で `AsyncOpenAI` を初期化 |
| `generate` | `async (…) → str` | `ChatCompletion` で回答を生成。`response.choices[0].message.content or ""` で None ガード |

### 2.8 `app/adapters/vector_store/supabase.py`

**`SupabaseVectorStore` クラス（`BaseVectorStore` の実装）**

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `__init__` | `(pool: asyncpg.Pool)` | コネクションプールを保持 |
| `search` | `async (query_embedding, top_k, min_similarity, min_authority_score) → list[SearchResult]` | サブクエリで IVFFlat を活用したコサイン類似度検索 |
| `save_chat_history` | `async (session_id, query, response, sources) → None` | `session_id` が None の場合はスキップ。`sources` は `json.dumps` で JSONB に保存 |

## 3. バッチモジュール仕様

### 3.1 `batch/ingest.py`

| 関数 | シグネチャ | 説明 |
|---|---|---|
| `main()` | `() → None` | エントリポイント。環境変数チェック → sources.yaml 読み込み → 各ソースを処理 → サマリー出力 |
| `load_sources(path)` | `(Path) → list[SourceConfig]` | YAML を読み込み `SourceConfig` のリストを返す |
| `extract_text(source)` | `(SourceConfig) → str` | `file_type` に応じて適切な extractor を呼び出す |

### 3.2 `batch/chunker.py`

| 関数 | シグネチャ | 説明 |
|---|---|---|
| `count_tokens(text)` | `(str) → int` | tiktoken（cl100k_base）でトークン数を計算。未インストール時は `max(1, len(text)//2)` |
| `split_text(text, chunk_size=500, overlap=50)` | `(str, int, int) → list[str]` | 段落単位で結合し chunk_size を超えたら確定。長い段落は `_split_long_text` で処理 |
| `_split_long_text(text, chunk_size, overlap)` | `(str, int, int) → list[str]` | 文末記号（`。.!?`）で分割し overlap を保持 |
| `_tail(text, overlap)` | `(str, int) → str` | バイナリサーチで末尾 overlap トークン分の文字列を取り出す |

### 3.3 `batch/embedder.py`

**`EmbeddingBatchClient` クラス**

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `__init__` | `(api_key, model="text-embedding-3-small")` | 同期 `OpenAI` クライアントを初期化 |
| `embed_many` | `(texts, batch_size=100) → list[list[float]]` | 100 件ごとにバッチ処理。失敗時は指数バックオフ（1s・2s）でリトライ最大 3 回 |

### 3.4 `batch/db.py`

**`SourceConfig` データクラス（frozen=True）**

| フィールド | 型 | 説明 |
|---|---|---|
| `path` | `str` | ファイルパス |
| `title` | `str` | ドキュメントタイトル |
| `source_url` | `str` | 一意な URL（upsert キー） |
| `file_type` | `str` | `"markdown"` or `"pdf"` |
| `organization` | `str \| None` | 発行元組織名 |
| `authority_score` | `float` | 信頼性スコア（0〜1） |
| `content_date` | `str \| None` | コンテンツ作成日（YYYY-MM-DD） |

**`IngestRepository` クラス**

| メソッド | シグネチャ | 説明 |
|---|---|---|
| `upsert_document_with_chunks` | `async (source, chunks, embeddings, token_counts) → str` | `source_url` で documents を upsert → document_chunks を DELETE → 一括 insert。トランザクション内で実行 |

## 4. Pydantic スキーマ仕様（`app/models/schemas.py`）

| スキーマ | フィールド | バリデーション |
|---|---|---|
| `Source` | `document_id`, `title`, `source_url`, `organization?`, `authority_score`, `chunk_index`, `similarity` | `authority_score: ge=0, le=1` / `similarity: ge=-1, le=1` / `chunk_index: ge=0` |
| `SearchResult` | `Source` + `content` | — |
| `ChatRequest` | `query`, `session_id?`, `top_k?` | `query: min_length=1, max_length=500` / `top_k: ge=1, le=20`（`None` 時は設定値を使用） |
| `ChatResponse` | `answer`, `sources`, `session_id?` | — |
| `SearchRequest` | `query`, `top_k?`, `min_similarity`, `min_authority_score` | `min_similarity: ge=-1, le=1` / `min_authority_score: ge=0, le=1` |
| `SearchResponse` | `results`, `total` | — |
| `HealthResponse` | `status`, `version`, `services` | — |
