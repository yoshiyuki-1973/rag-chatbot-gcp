# 02 基本設計書

本ドキュメントは、RAG チャットボットの「何を作るか（外部仕様）」を定義する。
詳細な内部設計は [04_詳細設計書.md](04_詳細設計書.md) を参照すること。

---

## 1. システム概要

### 1.1 目的

スポーツに関する公開データ（Markdown・PDF）を取り込み、ユーザーの質問に **出典付きで回答する RAG（Retrieval-Augmented Generation）チャットボット** を構築する。

LLM・ベクトルストアを差し替え可能な設計とし、RAG システムの設計力・実装力を示すポートフォリオ作品として公開する。

### 1.2 スコープ

| 区分 | 内容 |
|---|---|
| **対象** | スポーツに関するテキスト質問への出典付き回答 |
| **対象** | Markdown・PDF 形式 of 公開データの取り込み |
| **対象外** | 画像・動画・音声データの取り込み |
| **対象外** | リアルタイムデータ取得（外部 API クロール等） |
| **対象外** | ユーザー認証・アカウント管理（MVP では未実装） |

### 1.3 想定ユーザー

| ユーザー | 目的 |
|---|---|
| エンドユーザー | スポーツに関する質問を入力し、信頼できる出典に基づく回答を得る |
| ポートフォリオ閲覧者 | RAG システムの実装例を評価する採用担当者・エンジニア |
| システム管理者（開発者） | データ投入・運用を行う |

## 2. 機能一覧

### 2.1 ユーザー向け機能

| 機能ID | 機能名 | 概要 | 優先度 |
|---|---|---|---|
| F-01 | チャット質問 | 自然言語で質問を入力し、出典付きの回答を得る | 必須 |
| F-02 | 出典表示 | 回答に使用したドキュメントのタイトル・URL・信頼性スコアを表示する | 必須 |
| F-03 | ドキュメント検索 | クエリに関連するドキュメントチャンクをベクトル検索で取得する | 必須 |
| F-04 | 会話履歴表示 | セッション内の過去の質問・回答をブラウザ上に表示する | 必須 |
| F-05 | 会話履歴保持 | セッション識別子を使い、会話履歴を DB に永続化する | 任意 |

### 2.2 管理者向け機能

| 機能ID | 機能名 | 概要 | 優先度 |
|---|---|---|---|
| F-06 | ドキュメント取り込み | Markdown・PDF をバッチ処理でベクトル DB に登録する | 必須 |
| F-07 | 冪等な再取り込み | 同一ドキュメントを再取り込みした場合に既存データを更新する | 必須 |

### 2.3 システム機能

| 機能ID | 機能名 | 概要 |
|---|---|---|
| F-08 | ヘルスチェック | API サーバー・DB・LLM の死活確認エンドポイントを提供する |
| F-09 | エラーハンドリング | LLM・DB・Embedding API のエラーを適切にハンドリングしユーザーに通知する |

## 4. データ概要設計

### 4.1 エンティティ一覧

| エンティティ | 概要 | 主な属性 |
|---|---|---|
| Document（ドキュメント） | 取り込んだ文書のメタ情報 | タイトル・URL・発行組織・信頼性スコア |
| DocumentChunk（チャンク） | 文書を分割した断片とベクトル | 本文テキスト・embedding ベクトル・チャンク番号 |
| ChatHistory（会話履歴） | ユーザーの質問・回答・参照出典 | セッションID・クエリ・回答・出典 JSON |

### 4.2 エンティティ関係

```
Document (1) ─────────── (N) DocumentChunk
                               └── embedding: vector(768)

ChatHistory ── sources フィールド（JSONB）で Document を参照
```

### 4.3 主要データ仕様

| データ | 形式 | 説明 |
|---|---|---|
| embedding ベクトル | `VECTOR(768)` | Vertex AI / Gemini API `gemini-embedding-001` の出力 |
| authority_score | `NUMERIC(3,2)` 0.00〜1.00 | 情報ソースの信頼性スコア |
| sources | `JSONB 配列` | チャンク・類似度・出典情報を JSON で保存 |

## 5. 外部システム連携

### 5.1 連携システム一覧

| 外部システム | 用途 | 通信方式 | 認証 |
|---|---|---|---|
| Vertex AI / Gemini API | テキスト生成 (LLM) / ベクトル変換 | HTTPS REST / SDK | IAM / API キー |
| Cloud SQL for PostgreSQL | DB・ベクトル検索 | TCP（PostgreSQL プロトコル） | 接続文字列 / IAM データベース認証 |

### 5.2 外部 API 依存関係

```
バックエンド ──→ Vertex AI / Gemini API     テキスト生成（/chat 時）
バックエンド ──→ Vertex AI / Gemini API     クエリのベクトル化（/chat・/search 時）
バッチ       ──→ Vertex AI / Gemini API     チャンクの一括ベクトル化（Ingest 時）
バックエンド ──→ Cloud SQL                  チャンク検索・履歴保存
バッチ       ──→ Cloud SQL                  ドキュメント・チャンク登録
```

### 5.3 障害時の影響

| 外部システム障害 | ユーザー影響 | システムの挙動 |
|---|---|---|
| Vertex AI / Gemini API | チャット・検索不可 | HTTP 500（LLM_ERROR または SEARCH_ERROR）を返す |
| Cloud SQL | 全機能停止 | HTTP 500（SEARCH_ERROR）を返す |

## 6. 非機能要件

### 6.1 性能要件

| 項目 | 目標値 | 備考 |
|---|---|---|
| チャット応答時間 | 10 秒以内 | LLM の応答時間に大きく依存 |
| 検索応答時間 | 3 秒以内 | pgvector ANN 検索 |
| バッチ処理速度 | 100 チャンク/分以上 | API のレート制限に依存 |

### 6.2 可用性要件

| 項目 | 要件 |
|---|---|
| SLA | ポートフォリオ用途のため規定しない |
| Cloud Run | 従量課金 / min-instances 設定によるコールドスタート制御 |
| Cloud SQL | 稼働状況に応じた常時起動 (db-f1-micro) |

### 6.3 セキュリティ要件

| 項目 | 要件 |
|---|---|
| API キー / 認証情報管理 | 環境変数または GCP IAM サービスアカウントで管理し、コードにハードコードしない |
| CORS | フロントエンドのオリジンのみ許可（本番環境） |
| 入力バリデーション | Pydantic でリクエストボディを検証（query: 1〜500 文字、top_k: 1〜20） |
| スタックトレース | レスポンスに含めない（ログのみ出力） |
| 認証 | MVP では未実装 |

### 6.4 拡張性要件

| 拡張項目 | 設計方針 |
|---|---|
| LLM 差し替え | `BaseLLMClient` 抽象層で吸収。環境変数 `LLM_PROVIDER` で切り替え |
| ベクトル DB 差し替え | `BaseVectorStore` 抽象層で吸収。Adapter 追加のみで対応可 |
| データソース追加 | `batch/extractors/` に新パーサーを追加するだけで対応可 |

### 6.5 コスト要件

| サービス | プラン | 月額コスト |
|---|---|---|
| Cloud Run (フロント) | 従量課金 | 無料枠適用によりほぼ $0/月想定 |
| Cloud Run (バック) | 従量課金 | 無料枠適用によりほぼ $0/月想定 (最小インスタンス0時) |
| Cloud SQL for PostgreSQL | db-f1-micro | 約 $11.20 / 月（約 1,750 円）/ 台 |
| Vertex AI / Gemini API | 従量課金 | ポートフォリオ用途（月1,000回チャット）で約 $0.56 / 月（約 90 円）想定 |

## 7. 制約条件・前提条件

| 区分 | 内容 |
|---|---|
| **制約** | LLM は Gemini（Google）を使用する（Vertex AI / Gemini API を経由） |
| **制約** | ベクトル DB は Cloud SQL for PostgreSQL（pgvector）を使用する |
| **制約** | バッチ処理はローカル Docker で実行する（必要に応じて Cloud Run Jobs 等への拡張可） |
| **制約** | データソースはスポーツに関する公開データに限定する |
| **前提** | GCP アカウントおよび Google AI Studio アカウントが取得済みであること |
| **前提** | Docker Desktop がローカルにインストール済みであること |
